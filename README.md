# Shumonin Bot

一個以 AI 驅動的 Telegram 羣組反垃圾訊息機械人。新成員加入後，機械人會透過多輪對話進行問答，由 AI 判定是否符合入羣資格，不符合則自動踢出或封禁。此外，羣組成員可以舉報可疑訊息，由 AI 判定是否為垃圾內容並自動處理。

## 它能做什麼

**入羣驗證**

新成員加入後，機械人在羣組中向其提問，成員需在羣組中回答。AI 會閱讀對話歷程，必要時追問，直至判定通過或失敗，失敗或超時則踢出。同一用戶累計失敗多次後會被永久封禁。

機械人支援管理員以自然語言描述評分標準，例如指定某些問題必須給出實質性正確回答，不接受模糊帶過等回應。

**名稱預篩**

新成員加入時，機械人會先檢查其顯示名稱和用戶名。名稱含有廣告、色情、詐騙等明顯異常特徵的用戶，在進入驗證流程前即被踢出。

**垃圾訊息舉報**

羣組成員 reply 某條訊息並發送 `/report`，機械人會將該訊息送 AI 判定。確認為垃圾內容則刪除訊息並封禁發送者，判定為正常則靜默忽略。無論結果如何，機械人均不發送任何通知，避免打擾羣組。舉報管理員的訊息會被直接刪除。

**防濫用機制**

- 待驗證成員回答超過 50 字，直接踢出
- 嘗試操控 AI（如要求忽略指令、扮演角色、寫程式等），直接踢出
- 驗證超時（預設 2 分鐘）視同失敗，計入失敗次數

## 準備工作

### 1. 建立機械人

前往 Telegram 的 [@BotFather](https://t.me/BotFather)，發送 `/newbot` 並按指示建立機械人，取得 Bot Token。

### 2. 關閉 Privacy Mode

機械人需要讀取羣組中所有訊息才能正常運作。預設情況下，Telegram 機械人只能讀取 `/` 開頭的指令，必須手動關閉此限制：

1. 在 BotFather 發送 `/mybots`
2. 選擇你的機械人
3. 進入 **Bot Settings → Group Privacy → Turn off**

未完成此步驟，機械人將無法讀取成員的答題訊息。

### 3. 將機械人加入羣組並授予管理員權限

機械人需要以下管理員權限才能正常運作：

- 限制成員
- 踢出成員
- 封禁成員
- 刪除訊息

## 配置與啟動

在專案根目錄建立 `.env` 文件：

```env
TELEGRAM_BOT_TOKEN=你的機械人Token
OPENAI_API_KEY=你的OpenAI金鑰
OPENROUTER_API_KEY=你的OpenRouter金鑰  # 選填，作為備用供應商
ALLOWED_CHAT_IDS=-100123456789,-100987654321
DB_PATH=bot.db
```

`ALLOWED_CHAT_IDS` 填入機械人允許服務的羣組 ID，多個羣組以逗號分隔。機械人只會回應白名單內的羣組，其他羣組的事件一律忽略。

取得羣組 ID 的方式：將 [@userinfobot](https://t.me/userinfobot) 加入羣組，它會告訴你羣組 ID（通常以 `-100` 開頭）。

安裝依賴並啟動：

```bash
pip install -r requirements.txt
python main.py
```

## 管理員設定

機械人在羣組中透過指令設定，所有指令只有羣組管理員可以使用。

### 首次設定

發送 `/setup`，機械人會逐步引導你輸入：

1. **驗證問題** — 新成員加入後機械人會提出的問題。例如：

   ```
   RIME 是什麼，你爲什麼要加這個群？
   ```

2. **評分標準** — 用自然語言告訴 AI 如何評判回答。你可以指定哪些問題必須嚴格作答，哪些問題只需言之成理即可。例如：

   ```
   針對「RIME 是什麼」這一問題，必須答出「輸入法」、「輸入法引擎」、「輸入法框架」、「IME」、「keyboard」等內容。

   不需要非常具體地說明 RIME 是什麼。例如，不需要包括它的功能或特性。

   不可以回答「RIME 是 rime」等顯而易見、不懂裝懂的內容。
   宣揚資歷，比如「我用了很久了」；或者故意賣萌，比如「哎呀，我可以學」之類迴避問題的答案一律不接受。

   追問一次；若還是答錯，踢出。

   「為何加入」比較寬鬆，只需要一個合理的答案，說明是感興趣，或者為了學習或討論即可。
   ```

3. **超時時間** — 新成員需在此時間內完成驗證，否則視同失敗。單位為分鐘，預設 2 分鐘。

設定完成後確認儲存，機械人即開始對新成員進行驗證。

### 修改設定

日後可單獨修改任一項目，無需重新完整設定：

- `/setquestion` — 修改驗證問題
- `/setexpected` — 修改評分標準
- `/settimeout` — 修改超時時間

### 其他指令

| 指令 | 說明 |
|------|------|
| `/settings` | 查看當前羣組的所有設定 |
| `/unban <user_id>` | 解除機械人對某用戶的封禁記錄，使其可以重新加羣並重新計算失敗次數 |
| `/status` | 確認機械人是否具備所有必要的管理員權限 |
| `/cancel` | 取消當前進行中的設定流程 |
| `/help` | 顯示所有指令說明 |

### 關於 `/unban`

Telegram 原生的解封操作不會重置機械人內部的封禁記錄。如果你希望被封禁的用戶重新加入，除了在 Telegram 端解封外，還需發送 `/unban <user_id>` 令機械人重置其失敗次數，否則用戶一旦重新加入，機械人會再次封禁他。

## 驗證流程說明（供管理員了解）

1. 新成員加入，機械人立即限制其發言
2. 機械人先檢查其顯示名稱，異常則直接踢出
3. 名稱正常則在羣組中發出驗證問題，並 @ 該成員
4. 成員在羣組中直接回答，機械人每次收到回答都送 AI 判定
5. AI 可能追問（`continue`）、通過（`pass`）或踢出（`kick`）
6. 通過後限制解除；失敗或超時則踢出，並累計失敗次數
7. 累計 3 次失敗後永久封禁

整個對話過程對其他羣組成員可見，可以令新成員在公開環境下展示對羣組主題的了解。
